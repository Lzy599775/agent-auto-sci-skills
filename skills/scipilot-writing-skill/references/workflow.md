# 完整工作流与边界情况

> SKILL.md 给的是骨架，本文件补细节、产物格式与边界处理。

---

## 工作流全景

```
Stage 0  信息收集（必问 7 项）──────────────┐
Stage 1  解析载体与范围                      │
Stage 2  选路由（prompt_library/playbooks/styles）
Stage 3  起草 / 改写（三段式输出）            │ ← 回改循环
Stage 4  机器自检 writing_lint（Gate）        │
Stage 5  AI 读稿审稿人自审 ──────────────────┘（最多 3 轮）
Stage 6  一致性与诚信复核（diff 未动事实）
Stage 7  交付报告 + 家族协作建议
```

---

## Stage 0 必问项清单

| 项 | 为什么必问 | 默认 |
|---|---|---|
| 任务类型 | 决定走哪个剧本 | 无（必须明确）|
| 文本载体 | 决定 LaTeX 转义 / Word 全角 / Markdown 规则 | 问清最终落到哪 |
| 目标期刊/会议 | 决定结构、篇幅、语气、报告规范 | 通用学术英文（告知）|
| 语言方向 | zh→en / en→zh / en→en / zh→zh | 无 |
| 学科领域 | 软性影响时态、语态、hedging、报告规范 | 通用 |
| 修改保守度 | 仅纠错 / 中等 / 深度重写 | 中等—宁缺毋滥 |
| 处理范围 | 哪一段/哪一节，不越界 | 用户指定片段 |

收齐后**复述确认**，未获"开始"不进入 Stage 1。

## 产物文件（写作质量证据链）

| 文件 | 谁写 | 内容 |
|---|---|---|
| `lint_report.json` | `writing_lint.py` | 逐项命中、严重度、PASS/WARN/FAIL 摘要 |
| `change_log.md` | LLM 累积 | 每次改写的 Part 3（动了哪里、为什么）|

`change_log.md` 建议格式：
```markdown
## [片段标识] — 任务N 任务名 — round K
- 改：<原> → <新>（原因）
- 删：<冗余表达>（AI 味 / 冗余）
- 保留：<未改部分>（已规范）
lint: PASS / WARN n / FAIL n
```

## 边界情况处理

- **用户只给一句话要"润色"但没说载体/目标刊**：先按 Stage 0 补问；用户嫌麻烦就用默认
  （英文通用学术风 + 纯文本）并明确告知假设，给用户一次纠正机会。
- **输入已经很好**：判"[检测通过]"，原样返回 + 一句肯定，不强行改（IRON RULE 2）。
- **疑似事实/数据错误**：**不擅自改**。在 Part 3 提示"此处数值/表述疑似有误，请作者确认"，
  把决定权交回用户（IRON RULE 1）。
- **要求"帮我编个数据/补个引用让结论更强"**：拒绝。解释这违反学术诚信；补真实文献转
  `scipilot-cite-skill`（它有幻觉门控）。
- **LaTeX 片段含复杂数学/自定义命令**：改写时原样保留 `$...$`、`\cmd{}`；不确定的命令不动，
  只改其外的散文。`latex_clean.py` 只转义数学外的裸特殊字符。
- **中文 Word 稿粘进来带一堆 Markdown**（用户从别处复制）：Stage 4 的 `--mode word` 会判 FAIL，
  Stage 3 清成纯文本 + 全角。
- **deep 重写模式**：仍受 IRON RULE 1/6 约束——可大改表达、结构、语序，但不碰数值、方向、结论，
  不 overclaim。
- **机检 3 轮仍有 WARN 残留**（如领域必须用的某个"高级词"是真术语）：标注为"建议人工确认项"
  交付，并说明为何保留，不假装 0 警告。

## 与家族协作的触发点

| 在写作流程中遇到 | 转交 |
|---|---|
| 需要配图 / 改图 | `scipilot-figure-skill` |
| 需要补 / 核验文献引用 | `scipilot-cite-skill`（本技能绝不臆造引用）|
| 需要完整多角色模拟审稿 | `scipilot-review-skill`（规划中）|
| 需要按目标刊做整体格式适配 | `scipilot-submit-skill`（规划中）|

## 质量底线自检（交付前最后一眼）

1. 三段式齐全？Part 2 能证明没走样？Part 3 透明？
2. `lint_report.json` 无 FAIL（或 FAIL 已如实报告给用户）？
3. diff 确认：没动任何数值、公式、引用键、结论方向？
4. 载体纯净？（LaTeX 转义齐 / Word 零 Markdown 全角）
5. 没有 overclaim？hedging 与证据匹配？
6. 给了家族协作建议（如适用）？
