技能定位

用于把可验证任务变成长时段自动迭代流程，核心是 modify -> verify -> keep/discard -> repeat。

适用场景

  • 代码或数据处理脚本需要持续修复
  • 需要优化可量化指标
  • 需要安全审计或 ship-readiness gate
  • 需要前台或后台长期运行

使用方法

  • 只有在目标和验证方式明确时启用
  • 先扫描仓库并确认 run mode
  • 每轮只做一个聚焦改变
  • 每轮必须机械验证并记录结果

包含内容

  • loop/plan/debug/fix/security/ship/exec 模式
  • runtime invariants
  • 结果日志
  • session resume
  • foreground/background 运行协议

领域化优化

  • 适合用于文献库索引脚本、GIS/ML 分析脚本、图件生成脚本和数据清洗流程的长期修复
  • 不适合开放式论文润色，除非存在可验证指标

质量检查

  • 不能用于普通问答
  • 不能覆盖用户未授权修改
  • run artifacts 不应提交
  • 连续失败时要 pivot 而不是硬跑
