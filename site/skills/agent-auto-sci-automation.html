Automation / Checkpoint / Safety

agent-auto-sci-automation

科研自动化与长期记忆子 skill，用于把自动研究、论文摄入、任务恢复、失败复盘和 API 安全边界封装成可控流程。

Research automation subskill for source manifests, checkpoints, failure logs, task recovery, and API safety boundaries.

适合什么时候用

长期项目

当研究需要多轮检索、下载、阅读、分析和写作时，用它建立 manifest 和 checkpoint。

自动化工具接入

当接入外部 API、自动研究工具或本地知识库时，用它先定义密钥、缓存、日志和失败恢复边界。

典型输入

使用 agent-auto-sci-automation，为我的文献综述项目设计 source manifest、任务 checkpoint、失败日志和 API 配置安全规则。

Codex 部署与调用

安装整个仓库后重启 Codex。推荐调用方式：“使用 agent-auto-sci-automation，把这个科研任务变成可 checkpoint 的长期工作流。”

.\scripts\install.ps1

它会输出什么

  • source manifest 与任务状态结构。
  • 自动化流程的检查点、恢复点和失败记录格式。
  • API key、token、本地私有数据和公开仓库之间的边界。
  • 哪些自动化可以运行，哪些必须人工确认。

使用注意

它强调“可控自动化”。任何涉及密钥、付费数据库、私有 PDF 或未发表论文的内容，都应该留在本地私有项目中，不进入公开 skill。
