auto-sci-research

总领型自动科研 Agent skill。它是整个 Auto-sci-research 套件的入口，用来统筹地理学、体育学与城市健康研究中的任务路由、长期记忆、全流程写作、子 skill 调度和进化档案维护。

Umbrella research-agent skill for routing tasks, preserving reusable memory, selecting subskills, and maintaining the evolution archive.

适合什么时候用

跨阶段科研任务

当一个请求同时涉及文献、方法、GIS、数据分析、模型、写作或投稿时，先用它判断路线。

Skill 体系维护

当需要新增、合并、优化或审计子 skills 时，它负责检查是否重复、是否应该写入进化记录。

典型输入

使用 auto-sci-research，帮我判断“体育公园暴露与城市健康”的研究应该走系统综述、文献计量还是实证分析路线，并列出需要调用的子 skills。

Codex 部署与调用

git clone https://github.com/Lzy599775/agent-auto-sci-skills.git
cd agent-auto-sci-skills
.\scripts\install.ps1

安装后重启 Codex。推荐自然语言调用：“使用 auto-sci-research 规划这个科研任务”。旧触发词 agent-auto-sci 仍作为兼容关键词保留。

它会输出什么

  • 任务类型判断：文献、方法、空间、数据、模型、写作或 skill 进化。
  • 子 skill 调用路线和执行顺序。
  • 需要读取的项目记忆、参考文档和质量检查点。
  • 重大变动后的进化档案更新要求。

使用注意

它不是替代所有子 skill 的“大而全 prompt”。它的价值在于把任务分配到正确的专门模块，并保证长期项目不会失忆。
